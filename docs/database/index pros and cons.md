## Index 추가의 장단점은?
[인덱스] [1]는 데이터의 검색 속도를 향상시키기 위해서 별도의 자료 구조를 사용합니다. 따라서 검색 속도가 향상됩니다. 하지만 인덱스를 저장하기 위해 추가적인 저장 공간을 사용하고, 인덱스를 관리하는 추가 작업이 필요합니다.<br/>
인덱스를 관리하는 추가작업이란, MySQL의 InnoDB를 기준으로 인덱스는 B+트리 자료 구조를 사용합니다. 따라서 값이 정렬되는 순서를 고려하여 자료를 삽입, 삭제해야 합니다. 이 과정에서 인덱스의 트리를 재구성해야 할 수 있습니다.<br/>
