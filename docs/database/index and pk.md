## 인덱스와 PK의 관계
- [인덱스] [1]는 [PK] [2]에 의존합니다. 즉, 인덱스 별로 PK를 저장하고, PK를 통해 자료의 주소를 저장합니다.<br/>

### 인덱스가 PK에 의존하는 구조의 장점
- 레코드의 저장 위치가 바뀌어도 인덱스를 바꿀 필요가 없습니다.<br/>
- 연속되는 PK로 조회할 경우 랜덤 I/O가 아닌 순차 I/O를 사용하여 처리 속도가 더욱 빠릅니다.<br/>

### 인덱스가 PK에 의존하는 구조의 단점
- 인덱스를 통해 검색할 때 PK로 다시 한번 검색해야 하므로 처리 성능이 느립니다.<br/><br/>
