## PK란?
### PK
- 테이블 내에서 레코드의 위치를 유일하게 식별할 수 있는 칼럼 또는 그 조합.<br/>
- ``NOT NULL``, ``유니크`` 등의 특성을 갖는다.<br/>
- 레코드의 저장 위치를 결정한다.<br/>
- PK가 유사한 레코드들끼리 묶어서 저장하기 때문에 __클러스터 인덱스__ 라고도 불린다.<br/>

### PK의 자료 구조
- 인덱스와 마찬가지로 트리 구조로 관리된다.<br/>
- 리프 노드가 아닌 노드는 PK와 PK가 속한 자식 노드를 저장.<br/>
- 리프 노드는 레코드의 메모리 주소를 저장.<br/>
